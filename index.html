Node.js

¿Que es Node y como funciona?

Node.js es un entorno de ejecución JavaScript de código abierto y multiplataforma. Es una herramienta popular para casi cualquier tipo de proyecto. Node.js ejecuta el motor V8, el núcleo Google Chrome de JavaScript, pero lo hace fuera del navegador. Esto permite que Node.js tenga un gran rendimiento.

Ventajas de Node.js

Node.js tiene una ventaja única porque millones de desarrolladores de front end que escriben JavaScript para el navegador ahora pueden escribir el código del lado del servidor o backend sin necesidad de aprender un lenguaje completamente diferente. Esto sería el trabajo de un desarrollador full stack.

Otra ventaja de Node.js en la cual realmente brilla es en la creación de aplicaciones de red rápidas y escalables, ya que es capaz de manejar una gran cantidad de conexiones simultáneas con un alto rendimiento, lo que equivale a una alta escalabilidad. Con el término escalabilidad se hace referencia a la capacidad de Node.js de realizar muchas conexiones de manera simultánea sin que tenga que leer el código línea a línea, ni abrir múltiples procesos.

Asincronia en Node.js

Node está basado en JavaScript, por lo cual, usa solamente un hilo, no tiene varios en los que puede ir manejando la carga, en el caso del front-end tenemos las técnicas del async y await, las cuales nos permiten esperar el resultado, o hacer algo de forma asincrónica.
Node en cambio es asincrónico por defecto. Cuenta con un tipo de asincronía no bloqueante, lo cual significa que no se bloquea y espera a que una tarea termine.

Pongamos un ejemplo:

Tenemos un restaurante de comida asiática, el cual solo tiene un mesero, este tiene que ser lo suficientemente eficaz para atender a todas las mesas lo más rápido posible.
👇

El mesero toma nota del pedido de la Mesa 1

El mesero entrega el pedido de la Mesa 1 a la cocina

El mesero toma nota del pedido de la Mesa 2 mientras la cocina prepara la entrega 1

El mesero entrega el pedido de la Mesa 2 y recoge la comida de la Mesa 1

El mesero entrega la comida a la Mesa 1

El mesero recoge y entrega la comida a la Mesa 2

Se podrán imaginar lo poco eficiente que seria el mesero si esperase a que el pedido se cocinase antes de ir a buscar el siguiente.
Bueno, algo muy parecido hace Node. Solamente tiene un mesero, (en este caso es un hilo), el cual se encarga de todas las órdenes, pero lo hace de forma eficiente y asíncrona para que el programa que se encarga de recibir y responder a las órdenes no se quede esperando a la respuesta.

Ejemplos de Uso

Según Jeff Harrell, actualmente trabajando en Google pero con pasado como ingeniero de software de PayPal, al migrar desde Java a Node.js la eficiencia del desarrollo se duplicó (2 personas hicieron el trabajo de 5 personas en menos tiempo), el tamaño del código se redujo en un 33%, la cantidad de archivos se redujo en un 40% y se duplicaron la cantidad de request por segundo, aumentando en un 35% la velocidad de respuesta.

Node.js es utilizado por importantes compañías como: Netflix, Uber, Walmart, NASA, eBay y PayPal, entre otros.

Diferencias entre el navegador y Node

En el navegador, la mayor parte del tiempo lo que estás haciendo es interactuar con el DOM, u otras APIs de la plataforma web como las cookies. Esas no existen en Node.js. Por supuesto no tienes los objetos document, window y todos los demás objetos que proporciona el navegador.

En su defecto tenemos sus sustitutos que nos sirven para hacer cosas parecidas desde el backend

Por ejemplo el objeto console y sus metodos, son parte del navegador web, no de JavaScript como tal.
En la web es parte del objeto window.
En Node es parte del objeto global.
De todas formas no es necesario escribir window.console.log() para hacer un console.log sino que JavaScript lo interpreta automaticamente.

Node y los modulos

En el front-end tenemos el archivo HTML al que estamos modificando con nuestro archivo JS mediante el DOM.
Un ejemplo sería el proyecto individual, nosotros podemos usar la función getJSONData porque en el HTML ya agregamos previamente el script init.js y se cargo la función previamente.

En este caso, cuando escribimos nuestro código en cart.js, podemos utilizar las variables y funciones definidas en init.js porque este se carga antes que el.
En Node no es así. Como ya mencionamos, Node se ejecuta desde el servidor y no desde el cliente, así que estamos ejecutando directamente el archivo de JavaScript.

Node utiliza los módulos para ello. Nosotros podemos crear archivos JS para después exportar sus contenidos e importarlo en otro archivo.
Hay varias formas de hacer esto, la antigua forma y la nueva, vamos a explicar las dos porque ambas son relevantes. Vamos a hacerlo creando una situación para demostrar cómo se haría con ambas.
👇

Forma tradicional

Codigo de module.js
👇

Tenemos nuestro archivo index.js, el cual va a ser el que ejecutemos con Node.
Después tenemos el archivo module.js, el cual contiene la función la cual queremos ejecutar sin necesidad de tenerla en nuestro index.js.
Los archivos package.json y package-lock.json son los archivos de configuración de Node, una de las formas para exportar módulos necesita que modifiquemos esta configuración.

Tenemos una función llamada plataforma la cual nos retorna un string con la plataforma en la que estamos corriendo el código (process es un objeto de Node, el cual tiene varios usos pero uno de ellos es saber la plataforma en la que estamos corriendo nuestro archivo haciendo process.platform).

Nuestro archivo index.js esta configurado tal que asi 👉

En el codigo de index.js creamos una variable la cual va a ser donde se importe la función que escribimos en el otro archivo, esta variable va a ser igual a una función de Node.js la cual nos permite cargar módulos. A esta función le pasamos como parámetro donde se encuentra el archivo que queremos cargar, en este caso le pasamos como parámetro “./modulo”, el punto y el slash significa que empezamos a buscar a partir de donde esta el archivo que estamos corriendo con Node, en nuestro caso el archivo estaba en: C:\Users\(Nombre de usuario)\Desktop\pruebas entonces Node empezaría a buscar el archivo en: C:\Users\(Nombre de usuario)\Desktop\pruebas\modulo.js, Si nos damos cuenta al cargar un archivo JavaScript no necesitamos especificar cuál es su tipo de archivo, ya que Node por defecto lo carga como un archivo “.js”.

Después tenemos un console.log de lo que nos devuelve esta función. Para correr un archivo js con Node abrimos una terminal como por ejemplo la de git y escribimos “node (nombre de archivo)”.

Al correr nuestro archivo esto es lo que sucede: Nos dice que nuestra plataforma actual es win32 (ya que corrimos nuestro código en Windows).

Como podemos ver corrimos el archivo index.js, este cargo el módulo solicitado y pudimos usar una función de otro archivo en nuestro código.

Como ya mencionamos hay otra forma de importar otro código al index.js, esta usa las nuevas funciones agregadas con el tiempo a JavaScript.
Para empezar tenemos que agregar una configuración extra a Node desde el package.json 👇

El archivo package.json es un archivo de configuración que Node genera con cada proyecto, este es un objeto y lo que estamos interesados nosotros es en el type.
Por defecto este no aparece en el archivo sino que tenemos que agregarlo, si no lo agregamos este tendría el valor de commonjs, para nuestro caso el type tenemos que cambiarlo a module.
